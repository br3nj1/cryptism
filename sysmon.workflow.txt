# Sysmon Tuning Workflow

## Focus: Event ID 7 (ImageLoad) and Event ID 22 (DNS)

This workflow is designed specifically for:

* small advanced home labs
* PowerShell-heavy environments
* gaming + admin systems
* security tooling present
* long-term iterative tuning

The goal is:

```text id="cok8yt"
Keep the detections
Remove the entropy
```

NOT:

```text id="z7q5d0"
Remove the logs
```

That distinction is critical.

---

# Why These Two Events Matter

| Event       | Why Valuable                               | Why Dangerous     |
| ----------- | ------------------------------------------ | ----------------- |
| 7 ImageLoad | DLL sideloading, loaders, unsigned modules | Extreme volume    |
| 22 DNS      | C2, malware, beaconing                     | Browser explosion |

These two events:

* produce some of the BEST detections
* are also the fastest way to overwhelm storage and analysts

---

# Your First DNS Exclusions

```xml id="e4v4iu"
<DnsQuery onmatch="exclude">
```

# PHASE 2 — ImageLoad Tuning

This is the hard part.

---

# STEP 2 — Categorize DLL Loads

Split into:

| Category                     | Action          |
| ---------------------------- | --------------- |
| Microsoft signed system DLLs | Usually exclude |
| Browser sandbox DLLs         | Exclude         |
| GPU/vendor DLLs              | Usually exclude |
| Temp/AppData DLLs            | Keep            |
| Unsigned DLLs                | Keep            |
| User-writeable paths         | Keep            |
| Archive extraction paths     | Keep            |
| Rare DLLs                    | Investigate     |

---

# DO NOT DO THIS

Never:

```xml id="e9n7u8"
<ImageLoad onmatch="exclude">
    <Signed condition="is">true</Signed>
</ImageLoad>
```

Why?

Many malware families:

* use signed DLLs
* side-load signed binaries
* abuse vulnerable signed drivers

---

# Instead Use PATH-BASED TUNING

This is MUCH better.

# STEP 3 — Hunt Before Excluding

Critical rule:

```text id="jlwmck"
Never exclude before reviewing anomalies.
```

Common mistake:

* analysts tune too early
* remove actual detections

---

# Example Good Review Questions

## DLL Load

Ask:

* Is this unsigned?
* User-writeable path?
* Rare?
* Loaded by Office?
* Loaded by browser child process?
* Loaded by LOLBIN?
* Newly observed?
* Random filename?
* ADS involved?

---

# Example DNS Questions

Ask:

* Is PowerShell doing DNS?
* Is Office doing DNS?
* Is rundll32 making requests?
* Is DNS happening from Temp/AppData executables?
* DGA-like domains?
* Newly registered domains?
* Excessive TXT queries?
* Internal RFC1918 beaconing?

